 Implementation Plan: Neighbourhood Energy Simulation

 Context

 Build a prototype Node.js application that simulates electricity use/generation in a neighbourhood with 30 houses and 6 public EV chargers. The simulation must be
 deterministic (seeded PRNG), configurable via YAML, and include an animated browser-based UI with real-time updates.

 Tech Stack

 - Runtime: Node.js (no frameworks — pure http module)
 - 1 npm dependency: js-yaml (YAML config parsing — Node.js has no built-in YAML support; this is required by FR2.1)
 - Frontend: Single HTML file, vanilla JS, Chart.js loaded from CDN (no npm dependency)
 - Real-time updates: Server-Sent Events (SSE) — simpler than WebSockets, built into HTTP
 - Testing: Node.js built-in node:test runner (no test framework dependency)

 Key Design Decisions

 1. Seeded PRNG (mulberry32)

 10-line implementation. All randomness flows through a single seeded generator. Same seed + same config = identical results. No library needed.

 2. Simulation Loop

 - Uses setInterval with dynamic interval based on speed multiplier
 - Formula: realIntervalMs = (stepSizeMinutes * 60000) / speedMultiplier
 - At 60x: 1 simulated minute = 1 real second → 1 simulated hour in 60 real seconds
 - Each tick: advance clock → step all assets → record history → push SSE event

 3. SSE for Real-time UI

 - Server pushes SimulationState JSON to browser on every tick
 - Browser updates DOM and chart without polling
 - Simpler than WebSocket; no bidirectional communication needed

 4. Asset Behaviour Models (simplified, random-based)

 - Base consumption: Random walk around baseLoad ± variance
 - Heat pump: Random consumption within power limit (weather-aware later)
 - PV: Random generation within peak power, scaled by a simple time-of-day factor (daylight hours produce more)
 - Home EV charger: Random session start/stop; when active, draws constant charge power
 - Public EV charger: Independent random sessions, same power model

 5. Energy Accounting

 - cumulativeEnergy_kWh += currentPower_kW * deltaTimeHours on every step
 - PV generation is negative power; offsets house load first
 - Neighbourhood aggregate = sum of all house net power + public charger power

 6. UI Layout

 - Top bar: current date/time, season, speed controls (play/pause, 1x/10x/60x/3600x)
 - Main area: neighbourhood net power gauge + 24h line chart
 - Bottom: scrollable list of houses with per-asset cumulative energy
