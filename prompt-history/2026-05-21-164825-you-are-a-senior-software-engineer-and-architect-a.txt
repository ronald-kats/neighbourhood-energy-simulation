  Design Changes to Existing Classes

  - Asset.step(deltaHours) → Asset.step(deltaHours, weather) — breaking change
  anticipated in the deferred section of the original requirements
  - PV — production now driven by irradianceFactor from weather instead of
  random variation
  - HeatPump — consumption driven by temperature delta (indoor target − outdoor)
   instead of random variation
  - BaseHouseholdConsumption — optionally modulated by time-of-day and season
  - SimulationEngine — gains a WeatherModel; tick sequence inserts a
  weather-update step between clock advance and asset stepping
  - SimulationState — now carries currentWeather so the UI can display
  temperature, irradiance, cloud cover, and season

  Weather Model Details

  - Temperature: sinusoidal annual curve (peak July, trough January) + diurnal
  curve (peak 14:00, trough 04:00) + small deterministic jitter. Defaults to
  Dutch climate (~10 C mean, ~7 C amplitude).
  - Irradiance: computes solar elevation from latitude + day-of-year + hour;
  attenuated by cloud cover; zero at night.
  - Cloud cover: random walk with seasonal bias (winter cloudier, summer
  clearer), bounded to [0, 1], all deterministic via the seeded PRNG.
  - Season: derived directly from the simulated month.

  YAML Configuration

  A new optional weather: section allows overriding defaults for latitude,
  temperature amplitudes, cloud persistence, indoor target temperature, etc.
